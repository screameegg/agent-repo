# 知栈平台功能介绍

## 一句话定位

知栈是面向 AI Agent 的资产管理、技能同步、长期记忆、目标协作和迁移备份平台。它把分散在不同工具、不同模型、不同运行环境里的 Agent 能力沉淀为可管理、可审计、可迁移的数字资产，让 Agent 不只是一次性对话，而是能持续演进的工作伙伴。

## 平台解决什么问题

AI Agent 真正进入日常工作后，会遇到几个现实问题：

- 技能散落在本地提示词、脚本、文档和不同工具里，难以复用。
- Agent 换平台、换环境后，过去积累的经验、规则和目标容易丢失。
- 人类用户修改了 Agent 配置后，Agent 不一定知道，也不清楚何时应该重新学习。
- Agent 自己同步状态、记忆和目标时，如果没有边界，容易产生重复数据或错误确认。
- 团队想共享 Agent 能力，但缺少权限、审核、备份和版本管理机制。

知栈的目标是给 Agent 一个统一的资产层：技能放在 Skill Package，知识沉淀为记忆，工作方向拆成目标，配置变化通过事件通知，迁移通过备份完成。

## 核心能力

### Agent 管理

知栈 支持创建和管理多个 Agent 身份。每个 Agent 都可以维护名称、角色、职责描述、系统提示词、模型偏好、头像、技能数量、记忆数量、目标数量和同步状态。

对于外部 AI，平台提供 Agent Token 接入方式。Agent 可以通过 Token 自注册、绑定身份、同步自身状态、拉取平台配置，并在应用配置后确认事件。对于人类用户，前端提供 Agent 详情页，用来集中查看这个 Agent 当前拥有的技能、记忆、目标、同步状态和关联状态。

### Skill 能力库

知栈 把可复用能力定义为 Skill Package。一个 Skill 不只是一个提示词，而是一组完整文件树，可以包含：

- `SKILL.md`：技能入口、触发条件、使用边界和读取顺序。
- `references/`：较长的背景资料、API 文档、领域知识。
- `examples/`：请求样例、响应样例、配置样例。
- `scripts/`：可选脚本示例或工具调用说明。

Skill 可以设为私有、公开、草稿、发布或进入审核流程。用户可以把 Skill 挂载到指定 Agent，Agent 拉取配置时能读取完整文件树，而不是只拿到一段简单描述。这让团队可以把最佳实践、项目规范、工具说明和业务知识沉淀成可复用能力包。

### 记忆管理

Agent 在工作中会不断积累事实、偏好、流程、经验和项目背景。知栈 提供记忆管理能力，把这些内容结构化保存到平台中。

记忆可以带有类型、重要等级、来源和时间信息。人类用户可以手动注入关键记忆，Agent 也可以通过同步接口写入自己确认过的长期信息。这样 Agent 在不同会话、不同运行环境之间，仍然能继承关键上下文。

### 目标与任务步骤

知栈 支持为 Agent 维护执行目标。每个目标可以包含标题、描述、状态、优先级、截止时间和任务步骤。目标状态支持待处理、进行中、已完成、暂停、失败等工作流。

这让用户可以把“长期方向”拆成可跟踪的执行项。例如研发 Agent 可以有“完成模块重构”“修复同步 bug”“整理发布说明”等目标；每个目标下再拆分小步骤。Agent 执行后可以通过同步接口更新目标进度，人类用户也可以在前端直接编辑目标和步骤。

### 同步与 ack 闭环

知栈的同步机制把两类动作区分开：

- 平台侧或用户侧修改配置：例如用户挂载 Skill、卸载 Skill、编辑 Agent 身份、编辑记忆或目标。此类动作会生成 pending 配置事件，Agent 轮询 events 后拉取 config，应用完成后 ack。
- Agent 自己向平台同步状态：例如 Agent 上传自己的记忆、目标进度、技能快照。此类动作是 Agent 自身状态回传，不应该制造“自己确认自己”的 ack 循环。

同步写入前，平台要求 Agent 先拉取线上 config，读取 `syncRevision`，对比线上已有数据后再提交 `baseRevision` 和确认标记。这可以降低重复写入、误覆盖和并发冲突。

### Token 权限体系

Agent Token 支持细粒度权限，包括：

- Agent 自注册和同步。
- 配置读取和事件确认。
- Skill 读取和写入。
- 记忆读取和写入。
- 目标读取和写入。
- 备份导出。

团队可以按 Agent 的职责发放最小权限。只读 Agent 可以只拉取配置；同步 Agent 可以写入记忆和目标；迁移工具才需要备份导出权限。

### 备份、迁移与恢复

知栈 支持 Agent 资产导出和导入。备份可以包含 Agent 身份、技能挂载、Skill 文件树、记忆和目标。

这适合三类场景：

- 环境迁移：从测试环境迁移到生产环境。
- 平台迁移：从一个 Agent 运行平台迁移到另一个运行平台。
- 长期归档：保存某个阶段 Agent 的能力快照，便于回滚和复盘。

前端提供资产迁移入口，后端也提供 API 和 ZIP 结构，方便脚本化处理。

### 管理和审核

平台包含管理员能力，可以管理用户、公告和 Skill 审核。对于团队或企业内部使用，Skill 上架和公开可以进入审核流程，避免未确认的能力包被随意传播。

## 典型使用场景

### 研发 Agent 团队

研发团队可以把项目结构、代码规范、接口约定、调试流程和发布步骤沉淀为 Skill。Agent 每次接手任务前，从 知栈 拉取当前项目 Skill、长期记忆和执行目标，再根据目标步骤推进工作。

适合场景：

- 代码审查助手。
- 项目重构助手。
- 测试修复助手。
- 发布说明生成助手。
- 仓库迁移和依赖升级助手。

### 企业内部 AI 能力中台

企业可以把常用业务流程做成 Skill Package，由管理员审核后发布。不同部门或不同岗位的 Agent 按需挂载不同 Skill。平台通过 Token 权限控制读取和写入范围。

适合场景：

- 客服知识库 Agent。
- 销售资料整理 Agent。
- 内部流程助手。
- 数据分析助手。
- 运维排障助手。

### 多 Agent 协作网络

一个团队可能有多个 Agent：研发、测试、产品、运维、文档。知栈 可以分别管理它们的身份、技能、记忆和目标。每个 Agent 只读取自己被授权的配置，避免能力和上下文混乱。

适合场景：

- 多角色开发工作流。
- 多模型能力编排。
- 多项目 Agent 分组管理。
- Agent 生命周期管理。

### Agent 迁移和克隆

当一个 Agent 在旧环境中积累了大量 Skill、记忆和目标时，可以通过 知栈 导出完整备份，再导入到新环境。这样不需要从零配置，也能保留能力演进记录。

适合场景：

- 从本地实验迁移到服务器部署。
- 从个人 Agent 升级为团队共享 Agent。
- 为新项目复制一个已有 Agent 模板。
- 在故障后恢复 Agent 配置。

### 人机协作执行计划

用户可以在平台创建目标，拆分步骤，调整优先级。Agent 轮询到平台配置变更后执行任务，再把进度同步回平台。用户不需要追问“你现在做到了哪一步”，可以直接在目标列表查看状态。

适合场景：

- 长周期研发任务。
- 文档整理任务。
- 数据清洗任务。
- 运营内容计划。
- 项目交付清单。

## 产品价值

### 对个人用户

- 把自己常用的 Agent 配置沉淀下来。
- 换工具或换电脑时不用重新配置。
- 能看到 Agent 记住了什么、正在做什么、有哪些技能。
- 可以随时备份和恢复。

### 对团队

- 团队最佳实践可以变成可复用 Skill。
- 多个 Agent 能共享统一的能力标准。
- 管理员可以审核公开 Skill，降低误用风险。
- Agent 的目标和进度可以被看见、被调整、被追踪。

### 对企业

- 构建内部 AI 能力中台。
- 统一管理 Agent 资产、权限、配置、审核和迁移。
- 降低 AI 能力沉淀在个人电脑或单一平台里的风险。
- 为后续审计、版本治理和运行监控打基础。

## 与普通提示词管理的区别

普通提示词管理通常只保存一段文本。知栈 管理的是完整 Agent 资产：

- 不只是提示词，还有 Skill 文件树。
- 不只是知识库，还有结构化记忆。
- 不只是任务列表，还有可同步的目标步骤。
- 不只是复制粘贴，还有 Token、权限、事件、ack 和同步修订。
- 不只是导出文本，还有可迁移的 Agent 备份。

## 下一步优化场景

### 1. 同步预览 UI

把 Agent 即将写入的 Skill、记忆、目标差异展示在前端，让用户看到新增、更新、删除、跳过的内容，并选择同步范围。这样可以把当前 API 级别的同步预览要求产品化，降低误覆盖风险。

### 2. 目标小任务工作台

把目标步骤提升为更明确的任务工作台，支持步骤状态筛选、拖拽排序、执行日志、历史变更和完成率统计。这样目标不只是展示字段，而能成为人机协作的主工作区。

### 3. 配置变更审计

展示每一次 `config_changed` 的来源、原因、影响范围、创建时间、ack 时间和处理 Agent。这样用户可以追踪“是谁改了配置”“Agent 是否已应用”“为什么还待同步”。

### 4. Skill 版本治理

为 Skill Package 增加版本比较、变更日志、回滚和批量升级能力。用户可以看到某个 Agent 当前挂载哪个版本，也可以把多个 Agent 从旧版本升级到新版本。

### 5. Agent 运行健康度

基于最近轮询时间、pending 事件数量、pending 时长、同步失败次数和最近 ack 时间，展示 Agent 健康状态。这样用户能快速发现离线、权限错误或同步卡住的问题。

### 6. 场景化模板

提供研发助手、客服助手、文档助手、运维助手、数据分析助手等模板。模板包含默认 Skill、推荐权限、示例记忆和初始目标，帮助新用户快速上手。

### 7. 宣传落地页

以“Agent 资产管理平台”为第一屏定位，重点展示四个闭环：

- 技能库：把能力沉淀成 Skill Package。
- 记忆层：把长期上下文保存为结构化记忆。
- 目标层：把工作拆成可跟踪目标和步骤。
- 同步层：让 Agent 拉取配置、应用变更、回传进度。

## 对外宣传文案

### 标准版

知栈帮团队把 AI Agent 从一次性对话变成可管理、可迁移、可协作的数字资产。你可以为每个 Agent 配置专属技能、长期记忆和执行目标；Agent 通过 Token 安全读取平台配置，应用变更后确认事件，并把执行进度同步回平台。无论是研发助手、知识库助手，还是企业内部的多 Agent 协作网络，知栈 都提供统一的技能库、目标管理、同步协议和备份迁移能力。

### 短版

知栈是 AI Agent 的资产管理平台：统一管理技能、记忆、目标、同步和迁移，让 Agent 能持续学习、可靠协作，并在不同环境之间复用能力。

### 面向研发团队

把项目规范、调试经验、接口文档和执行计划沉淀到知栈。研发 Agent 每次工作前自动拉取最新技能、记忆和目标，完成后同步进度，团队可以清楚看到它会什么、记住什么、正在做什么。

### 面向企业

知栈为企业提供统一的 Agent 能力中台。通过 Skill 审核、Token 权限、配置同步、目标协作和备份迁移，企业可以把 AI 能力从个人实验升级为可治理、可复用、可审计的内部基础设施。
